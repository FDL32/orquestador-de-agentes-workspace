# AUDIT_WP-2026-147 - Graph context adapter

## Audit Focus

Verify that the destination-project graph-context adapter produces a compact, deterministic `## Project Context` block from existing `graphify-out` artifacts and that the controller can consume it without adding a heavy dependency stack.

## What Must Be True

- The adapter uses Python stdlib only.
- The adapter deterministically filters the existing graph output to the current ticket context.
- The adapter surfaces the relevant file inventory and immediate graph neighbors without bloating the prompt.
- The emitted context is compact enough to be useful as prompt context and does not exceed 30 lines.
- The `agent_controller.py` hook is optional and does not break existing ticket flow.
- The design remains lightweight and does not require a full knowledge-graph pipeline.

## Failure Modes

1. The adapter becomes LLM-heavy or introduces a new runtime dependency stack.
2. The generated context is too large to be practical as prompt context.
3. The filtering is too loose and yields noisy or misleading relationships.
4. The adapter fails when graphify output is absent or stale.
5. The controller hook becomes mandatory and breaks tickets when no graph output exists.

## Review Checklist

- Confirm the adapter is deterministic and stdlib-only.
- Confirm the output contains file inventory, immediate graph neighbors, and a compact summary.
- Confirm the controller hook is optional.
- Confirm the ticket still works when no `graphify-out` artifacts are present.
- Confirm the output is compact enough to seed future tickets with useful context.
