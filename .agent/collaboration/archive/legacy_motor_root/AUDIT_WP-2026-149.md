# AUDIT_WP-2026-149 - Bus-backed STATE projection sync

## Review Goal
Verify that the new sync path can keep `STATE.md` aligned with the bus without mutating the probe contract or introducing new drift.

## What Must Be True
- The sync path updates `STATE.md` from the bus-derived state when needed.
- The operation is idempotent.
- The probe remains read-only and still detects drift.
- Missing bus data and missing markdown state fall back safely.
- `STATE.md` uses the canonical plain `Estado actual: VALUE` line, and the controller write path matches that exact format.
- The probe validates the `# State - {ticket_id}` header before trusting the state line.
- The sync runs from the controller's main status path (`python .agent/agent_controller.py` with no flags) and from `--validate`.
- The implementation stays deterministic and stdlib-only.

## Failure Modes
1. The sync path rewrites `STATE.md` without using bus-derived state.
2. The sync is not idempotent and changes the file on repeated runs.
3. The probe stops being read-only or loses its diagnostic value.
4. Empty bus or missing `STATE.md` raises instead of degrading gracefully.
5. The markdown parser trusts the first `Estado actual:` line without checking the ticket header.
6. Tests only cover the happy path and miss drift, empty-bus, header-mismatch, or parse-failure cases.

## Review Checklist
- Confirm `STATE.md` is rewritten only when drift exists.
- Confirm a second sync call is a no-op.
- Confirm the probe still reports matched/drifted/bus_empty states.
- Confirm the parser validates `# State - {ticket_id}` before trusting `Estado actual:`.
- Confirm the implementation does not add dependencies.
- Confirm unit tests cover the sync path and the probe path independently.
