# PLAN_WP-2026-149 - Bus-backed STATE projection sync

## Goal
Turn `STATE.md` into an idempotent projection of the bus-derived state, using the existing deterministic probe logic as the foundation and avoiding manual markdown drift.

## Strategy
1. Reuse the deterministic bus/state comparison logic instead of re-implementing it.
2. Add a sync path that writes `STATE.md` only when the bus-derived state differs from the markdown projection, using the plain `Estado actual: VALUE` format as the canonical line.
3. Keep the read-only probe contract intact for diagnostics.
4. Preserve graceful fallback when the bus is empty or the markdown state cannot be parsed.
5. Call the sync path from the main controller status route (`python .agent/agent_controller.py` with no flags) and from `--validate`, so drift heals both during normal status reads and diagnostics.
6. Add focused tests for matched, drifted, empty-bus, missing-state, and ticket-header mismatch scenarios.

## Scope
- `.agent/agent_controller.py`
- `scripts/state_projection_sync.py`
- `scripts/state_projection_probe.py`
- `tests/unit/test_state_projection_probe.py`
- `tests/unit/test_state_projection_sync.py`

## Acceptance Shape
- `STATE.md` is updated from the bus-derived state when drift exists.
- The sync path is idempotent and safe to call repeatedly.
- The probe remains read-only and still reports drift.
- Empty bus and missing `STATE.md` degrade gracefully.
- The canonical `STATE.md` format is the plain `Estado actual: VALUE` line.
- The probe validates the `# State - {ticket_id}` header before trusting the state line.
- Tests cover both probe behavior and sync behavior.

## Notes
- No controller-wide workflow redesign is expected.
- No new dependencies are allowed.
- The goal is to eliminate state projection drift, not to change the ticket semantics.
