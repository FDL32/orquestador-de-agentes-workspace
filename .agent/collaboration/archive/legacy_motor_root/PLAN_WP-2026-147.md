# PLAN_WP-2026-147 - Graph context adapter

## Goal

Build a lightweight graph-context adapter that reads the existing `graphify-out/graph.json` and `GRAPH_REPORT.md` for a destination repository and turns them into a compact `## Project Context` block for the active ticket.

## Strategy

1. Read the existing graphify artifacts without rebuilding any graph.
2. Extract a compact ticket-scoped context block from `graphify-out/graph.json` plus `GRAPH_REPORT.md`.
3. Add a minimal `agent_controller.py` hook that can surface the context block when the graph output exists.
4. Keep the design intentionally smaller than a full knowledge-graph pipeline: no LLM-per-file analysis, no dashboard, no Node dependency.
5. Add focused tests for parsing, filtering, and compact output generation.

## Scope Notes

- This ticket is about deterministic context compression for destination projects.
- The adapter should stay in Python stdlib only.
- The hook into `agent_controller.py` should be optional and low-risk.
- The output should be compact enough to seed future tickets without flooding the prompt.
- The inspiration comes from graph-based project understanding tools, but the implementation here should remain lightweight and read-only.

## Acceptance Shape

- `scripts/graph_context.py` emits a deterministic ticket-scoped context block.
- The context includes file inventory, immediate graph neighbors, and a compact summary of the relevant corpus.
- `agent_controller.py` can surface a concise `## Project Context` summary when graphify artifacts are present.
- The adapter and tests run without extra dependencies.
- The ticket remains useful even if no `graphify-out` artifacts are present yet.
