# AUDIT_WP-2026-148 - Graph report summary enrichment

## Review Goal
Verify that the adapter now makes only the statistics section of `GRAPH_REPORT.md` useful without bloating the prompt or introducing new runtime dependencies.

## What Must Be True
- `GRAPH_REPORT.md` is consumed when available.
- Only the `## Estadísticas` section is consumed.
- The emitted context block stays at or below 30 lines.
- Report parsing is deterministic and stdlib-only.
- Missing or malformed reports fall back safely.
- The output remains ticket-scoped and concise.
- The line budget is pre-allocated by section instead of trimmed from the end.

## Failure Modes
1. The report is loaded but the statistics summary never appears in output, leaving dead code.
2. The file inventory leaks into the prompt instead of only the statistics section.
3. The context block grows beyond the 30-line cap.
4. Malformed report content raises instead of degrading gracefully.
5. The ticket becomes coupled to a new graph pipeline or external dependency.
6. Tests only cover the happy path and miss the absence/fallback path.

## Review Checklist
- Confirm the summary line derived from `GRAPH_REPORT.md` statistics appears in the emitted context when the file exists.
- Confirm the file inventory is not injected into the prompt.
- Confirm the no-report path still works.
- Confirm the line limit stays intact.
- Confirm the implementation remains read-only and deterministic.
- Confirm unit tests cover report-present, report-absent, and malformed-report behavior.
