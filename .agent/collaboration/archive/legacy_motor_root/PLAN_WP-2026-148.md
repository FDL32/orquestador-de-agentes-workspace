# PLAN_WP-2026-148 - Graph report summary enrichment

## Goal
Extend the graph context adapter so it consumes only the `## Estadísticas` section of `graphify-out/GRAPH_REPORT.md` and surfaces a concise report-backed summary inside the emitted `## Project Context` block. The change should keep the adapter read-only, deterministic, and compact.

## Strategy
1. Parse the existing `GRAPH_REPORT.md` with stdlib only and read only the statistics section.
2. Extract a compact one-line summary that is useful to the Builder but still short enough to preserve the 30-line cap.
3. Merge that summary into the current graph-context output with pre-allocated section budgets instead of reactive trimming.
4. Keep the implementation isolated to the adapter and its tests.
5. Reuse the existing report loading path rather than introducing a new graph pipeline.

## Scope
- `scripts/graph_context.py`
- `tests/unit/test_graph_context.py`

## Acceptance Shape
- The adapter uses `GRAPH_REPORT.md` when present.
- The adapter only consumes the `## Estadísticas` section and ignores the file inventory.
- The output remains deterministic and within the 30-line maximum.
- Missing or malformed report files degrade gracefully.
- The ticket-scoped context remains concise and useful.
- Tests cover report-present and report-absent behavior.

## Notes
- No controller hook changes are expected in this WP.
- No new dependencies are allowed.
- This is a refinement of the existing adapter, not a scanner rewrite.
