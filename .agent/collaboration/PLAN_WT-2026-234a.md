## 1. Contrato

Crear una barrera de portabilidad basada en el indice Git del `repo_motor`.
La barrera rechaza nombres versionados que incorporen IDs operativos:

```text
WT-YYYY-NNN          (hyphen, legacy)
WT-YYYY-NNNa         (hyphen + suffix)
WT_YYYY_NNN          (underscore, common in test filenames)
test_wt_YYYY_NNNa.py (underscore filename pattern)
WP-YYYY-NNN
PLAN-YYYY-NNN
```

El separador acepta `[_-]`, por ejemplo:
`(?i)(WT|WP|PLAN)[_-]\d{4}[_-]\d{3}[a-z]?`.

La deteccion se aplica al basename y reporta el path motor-relative. No inspecciona
contenido y no recorre archivos ignorados.
