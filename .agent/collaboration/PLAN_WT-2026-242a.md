# PLAN_WT-2026-242a

## 1. Contrato Tecnico

Este ticket corrige la ruta OpenCode del `review_bridge` para que use
`try-first` con `--format json` sobre el `manager_executable` real de la
ejecucion. El objetivo es impedir que una deteccion rota por `PATH` degrade una
review valida a `text_regex` y cambie el estado canonico del ticket.

El output minimo es:

- intento inicial con `--format json` usando el ejecutable real;
- fallback sin JSON solo ante evidencia explicita de flag no soportado;
- preservacion de la asimetria conservadora: `APPROVE` solo desde
  `json_final_answer`;
- tests gobernantes que cubran el bug observado.

## 3. Cambio Esperado

### Cambio minimo de motor

- La ruta OpenCode en `_run_opencode_review()` debe construir primero el
  comando con `--format json`.
- Ese intento debe usar el `manager_executable` real recibido por la llamada;
  no puede depender de `opencode.cmd` resuelto por `PATH`.
- `_supports_json_format` como campo de instancia calculado en `__init__` debe
  dejar de gobernar la decision JSON/no-JSON. Si el Builder lo elimina, debe
  eliminar tambien las referencias restantes. Si prefiere mantener una cache
  local por llamada dentro de `_run_opencode_review()`, `__init__` ya no puede
  invocar `_detect_json_format_support()`.
- Si el intento devuelve evidencia concreta de flag no soportado, el bridge
  debe relanzar sin `--format json`.
- Los patrones de fallback deben ser binarios y acotados a stderr/ayuda, por
  ejemplo:
  - `unknown flag`
  - `invalid option`
  - help banner / salida de ayuda del CLI
- `exit_code != 0` por si solo no habilita fallback.
- El output del intento fallido por flag no soportado se descarta por completo;
  el fallback debe arrancar como proceso nuevo con output limpio.
- Si el intento con JSON falla por timeout, auth, salida vacia o corrupcion no
  atribuible al flag, se trata como fallo real del intento, no como capability
  miss.

### Resultado observable

- un `manager_executable` valido fuera de `PATH` sigue usando JSON;
- una version real sin soporte para `--format json` cae a fallback de forma
  trazable;
- una review textual con `DECISION: APPROVE` sigue degradando a `INSPECT`.
