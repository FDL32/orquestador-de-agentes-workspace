# PLAN_WT-2026-240a

## 1. Contrato Tecnico

Este ticket corrige el bug residual dejado por `WT-2026-239a`: la rama
documental de `--pre-handoff` no debe auto-commitear ni exigir checkpoint en
`repo_motor`, pero tampoco puede dejar pasar cambios productivos sin commit.

El output minimo es:

- preservar el bypass documental de auto-commit, tag y checkpoint;
- reintroducir la verificacion de higiene del `repo_motor`;
- bloquear con `HANDOFF_BLOCKED` cuando `motor_uncommitted_productive()`
  detecte cambios productivos;
- dejar tests focales que cubran rama documental y no regresion de tickets
  `code`.

### Cambio minimo de motor

- En `_handle_pre_handoff()`, dentro de la rama para
  `documentation/research/analysis`, llamar primero a
  `motor_uncommitted_productive(motor_root)`.
- Si devuelve entradas productivas:
  - imprimir error explicito;
  - emitir `HANDOFF_BLOCKED`;
  - retornar `1`.
- Si no devuelve entradas:
  - continuar por el bypass documental ya existente;
  - no crear commit `pre-handoff checkpoint`;
  - no crear tag `checkpoint/review-<ticket>`;
  - seguir permitiendo arbol limpio en el repo activo.

- ticket documental con `repo_motor` sucio: `--pre-handoff` falla;
- ticket documental con `repo_motor` limpio: `--pre-handoff` pasa sin commit ni
  tag;
- ticket `code`: conserva la ruta actual sin regresion.
