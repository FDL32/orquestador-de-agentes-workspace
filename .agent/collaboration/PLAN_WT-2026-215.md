# PLAN_WT-2026-215

## Objetivo
Corregir que las operaciones git de evidencia/provenance del tooling de review y gates
usen `motor_root` como cwd, no `project_root`. Introducir un seam unico de resolucion
en `review_bridge.py` y migrar los call sites clasificados en `prepush_check.py` y
`session_closeout.py`.

## Root cause verificada
- `_resolve_motor_root()` y `_resolve_motor_controller()` ya delegan en `motor_link`
  (WT-2026-187), pero las funciones git de evidencia nunca se migraron.
- `cwd=self.project_root` en `_git_diff_stat`, `_build_diff_for_files_likely_touched`,
  `_resolve_review_base` y derivadas produce diffs del workspace, no del motor.
- Con la topologia actual (workspace y motor son repos hermanos con `.git` propios),
  el review packet refleja artefactos de colaboracion del workspace en vez de codigo real.

## Seam unico propuesto

```python
class ReviewBridge:
    def _motor_root_or_raise(self) -> Path:
    """Return motor_root for git evidence operations.

    Raises RuntimeError if motor link is not configured.
    """
    root = self._resolve_motor_root()
    if root is None:
        raise RuntimeError(
            "motor_root not resolvable: motor_destination_link.json missing or invalid. "
            "Cannot run git evidence operations."
        )
    return root
```

Cada funcion del grupo de evidencia/provenance llama `self._motor_root_or_raise()`
internamente; no hay threading de `motor_root` como parametro entre
`_resolve_review_base`, `_fallback_ticket_base_from_log`, `_get_current_git_head` o
`_compute_changed_files`.

Fallback para `prepush_check.py` y `session_closeout.py` (no tienen acceso a
`_resolve_motor_root`): llamar a `resolve_motor_root(project_root)` de `runtime.motor_link`
directamente; si devuelve None, registrar warning y saltarse la operacion git (no bloquear
por ausencia de link cuando el check no es el gate primario).

## Tests esperados

Archivo nuevo: `tests/test_motor_root_gates.py`

1. **test_diff_stat_uses_motor_root**: `_git_diff_stat` llama a `subprocess.run` con
   `cwd` igual a `motor_root`, no a `project_root`.
2. **test_check_review_packet_not_empty_with_motor_commits**: con motor que tiene commits
   reales y workspace sin commits de codigo, `check_review_packet_diff_empty` devuelve
   `False`.
3. **test_review_packet_uses_motor_diff_with_workspace_noise**: con ruido de
   colaboracion en el workspace y commits reales en el motor, el review packet usa
   diff/provenance del motor y no clasifica el caso principal como diff vacio.
4. **test_prepush_check_uses_motor_root**: `run_git_status_check` usa `cwd=motor_root`.
5. **test_session_closeout_git_clean_uses_motor_root**: `_step_git_clean` usa
   `cwd=motor_root`.
6. **test_motor_root_or_raise_no_link**: con link ausente, `_motor_root_or_raise` lanza
   `RuntimeError` con mensaje descriptivo.
7. **test_fallback_no_crash_prepush_without_link**: `prepush_check` sin link no hace crash,
   emite warning y continua.
8. **test_regression_cwd_project_root_breaks_check**: revert parcial de `_git_diff_stat`
   a `cwd=self.project_root` hace fallar `test_check_review_packet_not_empty_with_motor_commits`.
9. **test_out_of_scope_call_sites_unchanged**: call sites fuera de scope (repomix,
   review transport) conservan `cwd=self.project_root`.
