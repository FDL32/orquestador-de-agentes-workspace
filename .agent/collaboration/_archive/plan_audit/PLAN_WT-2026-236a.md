## 1b. Contexto y Problema

El protocolo `repo-compare` compara un repositorio GitHub contra el contexto local
usando AUDIT.md, Repomix y MCP GitHub. En el preflight humano de este ticket ya
aparecieron fricciones reales:

- MCP GitHub fallo con `Authentication Failed: Bad credentials`.
- `gh repo view stablyai/orca ...` fallo porque `gh` no esta autenticado.
- `.agent/runtime/audit/AUDIT.md` no existe en este `repo_destino`.
- `scripts/local_audit.py` existe en `repo_motor`, pero no esta instalado en
  `repo_destino`.

El Builder debe separar valor de producto e higiene del protocolo. Un fallback web
funcional no demuestra que `repo-compare` este sano.
