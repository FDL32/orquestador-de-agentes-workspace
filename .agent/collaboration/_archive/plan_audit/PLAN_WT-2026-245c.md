# PLAN WT-2026-245c - Centralizar patron canonico de ticket ID

## Estado
APPROVED

## Objetivo tecnico
Reducir la duplicacion del patron canonico de ticket ID en el motor, con una fuente canonica Python y una constante/helper equivalente en PowerShell.

## Alcance
- Crear `bus/ticket_id.py` con el patron canonico y helpers minimos de extraccion/match para consumidores Python.
- Sustituir regex inline duplicadas en `bus/review_bridge.py` y `bus/supervisor.py` por el helper canonico cuando sea viable.
- Revisar `agent_controller.py` y mantenerlo fuera del diff salvo evidencia nueva de duplicacion activa del mismo patron.
- Consolidar el patron equivalente en `scripts/launch_agent_terminals.ps1` usando `$script:TicketIdPattern` o helper local equivalente, sin acoplarlo a Python.
- Incluir `scripts/validate_ticket_prose.py` como superficie deliberada para soportar `AUDIT_<XXX>-*.md`.

## Secuencia de implementacion
1. Localizar todas las regex activas de ticket ID en Python dentro del scope.
2. Crear `bus/ticket_id.py` con:
   - patron canonico
   - helper de extraccion
   - helper de match si aporta claridad real
3. Reemplazar en `bus/review_bridge.py` y `bus/supervisor.py` las regex que duplican el contrato.
4. Revisar `.agent/agent_controller.py`; si no aporta valor tocarlo, dejarlo fuera del diff.
5. En `scripts/launch_agent_terminals.ps1`, sustituir duplicacion local por `$script:TicketIdPattern` o helper local del mismo patron.
6. Ajustar `scripts/validate_ticket_prose.py` para aceptar `AUDIT_[A-Z][A-Z][A-Z]-*.md` como contrato soportado.
7. Extender `tests/unit/test_launcher_powershell_syntax.py`; no crear un archivo de test nuevo paralelo.
8. Ejecutar tests y quality gates, registrar evidencia en `execution_log.md`.

## Criterios binarios de aceptacion
- Existe `bus/ticket_id.py` como fuente canonica Python.
- `bus/review_bridge.py` y `bus/supervisor.py` ya no repiten inline el patron central donde el helper aplica.
- `launch_agent_terminals.ps1` usa `$script:TicketIdPattern` o helper local y sigue aceptando `WP-*`, `WT-*` y prefijos de tres letras.
- `validate_ticket_prose.py` reconoce `AUDIT_<XXX>-*.md`.
- No aparece un modulo importable nuevo dentro de `prompts/`.

## Riesgos conocidos
- Riesgo de convertir una limpieza de regex en un refactor mas amplio de lo necesario.
- Riesgo de romper consumidores que requieren grupos de captura diferentes si se sobre-abstrae el helper Python.
- Riesgo de mezclar centralizacion Python con centralizacion PowerShell en un mismo mecanismo incorrecto.
