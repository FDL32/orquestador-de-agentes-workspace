# PLAN WT-2026-221a - Relaunch CEM

## Objetivo
Cerrar la brecha entre `CHANGES` y el siguiente Builder: solo relanzar cuando la
topologia `repo_motor`/`repo_destino` este verificada y entregar continuidad
operativa mediante una capsula fresh, evidence-linked y no acumulativa.

## Orden de trabajo recomendado
1. Seguir el camino real de relaunch en `bus/supervisor.py` y
   `scripts/launch_agent_terminals.ps1`.
2. Identificar un seam minimo de verificacion de topologia/root.
3. Implementar la barrera que bloquea relaunch invalido antes de abrir ventana.
4. Implementar la capsula fresh para relaunch post-`CHANGES`.
5. Colgar tests que reproduzcan el fallo seq 578 y el camino valido.

## Seams confirmados
- `bus/supervisor.py:_relaunch_builder()` - punto de entrada real del relaunch.
- `bus/supervisor.py:_verify_builder_start()` - origen de `verify_signal: none`.
- `bus/supervisor.py` emision de `BUILDER_RELAUNCH_ATTEMPTED` - payload observable del resultado.
- `scripts/launch_agent_terminals.ps1` resolucion de `AGENT_PROJECT_ROOT` - frontera de topologia/root del launcher.

## Reglas de ejecucion
- No proponer un framework grande antes de cerrar el bug real.
- Toda afirmacion sobre root, ticket activo o continuidad debe apoyarse en
  artefactos canonicos.
- Si el Builder necesita inferir algo, debe marcarlo como hipotesis y dejarlo
  fuera de los hechos verificados.
- Si aparece un cambio lateral, clasificarlo antes de tocarlo.

## Evidencia minima que debe dejar el Builder
- Descripcion corta del seam real de relaunch encontrado.
- Test o reproduccion del caso invalido tipo seq 578.
- Test del caso valido con root correcto.
- Nota de la capsula generada y de donde toma cada dato.
- Resultado de `ruff` y del subset de tests tocado.

## Criterio de READY_FOR_REVIEW
Solo cuando:
- el relaunch invalido queda bloqueado con mensaje observable;
- el relaunch valido entrega capsula fresh y evidence-linked;
- existe prueba gobernante del fallo original;
- el cambio sigue siendo minimo y no invade `WT-2026-221b` o `WT-2026-221c`.

Verificacion ejecutable minima:
- `pytest tests/ -k "relaunch" -q`
- `ruff check bus/supervisor.py scripts/launch_agent_terminals.ps1`
